    # 原地重建索引

    我们可以对现有索引的 Mapping 新增字段并指定新的 Analyzer，但是不能修改已有字段的 Analyzer。

    我们对于一个已经存在的索引添加子字段是不会影响现有的数据的，但是我们添加完子字段以后原有的文档并没有该子字段，不过没有关系，我们原地进行一次重建就好了，也就是将当前索引的数据重新替换一下当前索引，一般用于 Mapping 的修改，我们可以很方便的使用 update_by_query 接口来实现，如下：

    POST <index-name>/_update_by_query?conflicts=proceed

    conflicts=proceed : 有冲突不中断继续执行。
