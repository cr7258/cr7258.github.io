      # 206.反转链表

      # 题目描述

      给你单链表的头节点 head ，请你反转链表，并返回反转后的链表。 示例 1

      输入：head = [1,2,3,4,5]
输出：[5,4,3,2,1]

      示例 2：

      输入：head = [1,2]
输出：[2,1]

      # 解题思路

      # 解法一：非递归

      使用一个新的链表来记录反转后的结果，虽然是新链表，但是并不会增加空间复杂度，因为我们是改变原链表的指针。

      • 1.依次往后遍历原链表，每次用一个临时变量来保存 p.next。
      • 2.改变原链表的指针，指向新节点。
      • 3.用临时变量恢复原链表的下一个节点。

      # 解法二：递归

      • 1.通过递归函数，一直递归到链表的最后一个结点为止，此时，该结点就是反转成功后的头结点，是最终的返回结果。

      • 2.在递归函数中，让当前节点的下一个节点的 next 指针指向当前节点。

      • 3.在递归函数中，让当前节点的 next 指针指向 null。

      • 4.通过二三步的操作，已经让递归函数中的链表实现了局部反转，将结果（反转后的头结点）返回给上一层递归函数。

      # 代码

      # 解法一：非递归

      // 反转链表(LeetCode 206): https://leetcode-cn.com/problems/reverse-linked-list/
class Solution {
    public ListNode reverseList(ListNode head) {

        ListNode p = head;
        //虚拟头节点
        ListNode newNode = null;

        while(p != null){
            // 临时节点记录原链表的下一个节点
            ListNode tmp = p.next;
            // 改变节点指针，指向新的节点
            p.next = newNode;
            // 新节点后移
            newNode = p;
            // 通过临时节点恢复
            p = tmp;
        }
        return newNode;
    }
}

      # 解法二：递归

      class Solution {
    public ListNode reverseList(ListNode head) {

        // 寻找递归终止条件
        // 1、head 指向的结点为 null 
        // 2、head 指向的结点的下一个结点为 null 
        // 在这两种情况下，反转之后的结果还是它自己本身
        if( head == null || head.next == null)  return head;

        // 不断的通过递归调用，直到无法递归下去，递归的最小粒度是在最后一个节点
        // 因为到最后一个节点的时候，由于当前节点 head 的 next 节点是空，所以会直接返回 head
        ListNode cur = reverseList(head.next);

        // 比如原链表为 1 --> 2 --> 3 --> 4 --> 5
        // 第一次执行下面代码的时候，head 为 4，那么 head.next = 5
        // 那么 head.next.next 就是 5.next ，意思就是去设置 5 的下一个节点
        // 等号右侧为 head，意思就是设置 5 的下一个节点是 4

        // 这里出现了两个 next
        // 第一个 next 是「获取」 head 的下一节点
        // 第二个 next 是「设置」 当前节点的下一节点为等号右侧的值
        head.next.next = head;


        // head 原来的下一节点指向自己，所以 head 自己本身就不能再指向原来的下一节点了
        // 否则会发生无限循环
        head.next = null;

        // 我们把每次反转后的结果传递给上一层
        return cur;
    }
}
